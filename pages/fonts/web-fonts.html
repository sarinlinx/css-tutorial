Overview

Web Safe fonts are a good default, but can sometimes be limiting. Web Fonts is a solution to the generic Web Fonts.

Web fonts are a CSS feature that allows you to specify font files to be downloaded along with your website as it is accessed, meaning that any browser that supports web fonts can have exactly the fonts you specify available to it.

There are two types of Web Fonts:

  • Internal
  • External

Internal web fonts

Internal web fonts are files you download.

You can then include them in your site just like an image or CSS file.

Declare and link to it using @font-face

@font-face {
  font-family: 'Museo Sans';
  src: url(museo-sans.tff);//font file name
}

You must then add it to the font stack in CSS like you would add any other.

body {
  font-family: 'Museo Sans', Arial, sans-serif;
}

File formats

Most browsers support WOFF and WOFF2 fonts

@font-face {
  font-family: 'Museo Sans';
  src: url(museo-sans.woff2) format('woff');
}

External web fonts

External web fonts are 3rd party files you DO NOT download.

You just link to the font URL. Here are a few examples:

  • TypeKit
  • Google Fonts

Using Google fonts

  1. Click onthe + sign for a font
  2. It's saved in a drawer at the bottom
  3. Copy the link code into the of your doc BEFORE your custom css file
  4. Click on it from the drawer
  5. Next, copy the CSS code for the font and add the fonts to the body selector in your CSS file
  6. Back in Google fonts, click the 'Customize' link to choose which font-weights you want to load. Only load ones you need.
